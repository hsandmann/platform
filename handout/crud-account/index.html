    CRUD Account

    A. Requisito

    A fim do sistema possuir um controle de acesso, é conveniente a criação de um micro serviço Auth, que será responsável pelo cadastro de usuários do sistema. Tal micro serviço pode ter, ao menos, dois endpoints:

    1. Registro 

      POST /auth/register
      Request 
      {
    "name": "Antonio do Estudo",
    "email": "acme@insper.edu.br",
    "password": "123@"
}
      Response

      code body
      200 { "id": "45d16201-12a4-48bf-8c84-df768fdc4878", "role": "admin" }

    2. Autenticação 

      POST /auth/login
      Request 
      {
    "email": "acme@insper.edu.br",
    "password": "123@"
}
      Response

      code body
      200 { "id": "45d16201-12a4-48bf-8c84-df768fdc4878", "role": "admin" }
      401

    B. Fluxo Sugerido

    1. Registro 

      POST /auth/register


      sequenceDiagram
autonumber
actor User
User->>+Auth Server: register(RegisterIn)
Auth Server->>+Account Server: create(AccountIn)
Account Server->>-Auth Server: returns the new account (AccountOut)
Auth Server->>-User: returns 201

    2. Autenticação 

      POST /auth/login


      sequenceDiagram
autonumber
actor User
User->>+Auth Server: authenticate(CredentiaIn)
Auth Server->>+Account Server: login(LoginIn)
critical validated
    Account Server->>-Auth Server: returns the account
option denied
    Auth Server-->>User: unauthorized message
end  
Auth Server->>Auth Server: generates a token
Auth Server->>-User: returns the token
User->>User: stores the token to use for the next requests

    C. Modularização

    Crie dois projetos Mavens para cada micro serviço:

    • um projeto será a interface (compromisso de implementação do micro serviço), e;
    • o outro projeto será a implementação em si do serviço.

    A vantagem dessa abordagem é que a interface pode ser utilizada em outros projetos como uma biblioteca a ser consumida.

    Exemplo para o micro serviço Auth.

    classDiagram
  namespace Interface {
    class AuthController {
      <<interface>>
      register(RegisterIn)
      authenticate(CredentialIn)
    }
    class RegisterIn {
      <<record>>
      String name
      String email
      String password
    }
    class CredentialIn {
      <<record>>
      String email
      String password
    }
  }
  namespace Resource {
    class AuthResource {
      <<REST API>>
      -authService
    }
    class AuthService {
      <<service>>
      register(RegisterIn)
      authenticate(CredentialIn)
    }
  }
  AuthController <|-- AuthResource
  AuthResource o-- AuthService

    # docker compose up -d --build --force-recreate
version: '3.8'
name: store

  db-store:
    container_name: spkio-db-spkio
    image: postgres:latest
    command: postgres -c config_file=/etc/postgresql.conf

      - 5400:5432
    environment:
      - POSTGRES_USER=spkio
      - POSTGRES_PASSWORD=spkio
      - POSTGRES_DB=spkio

      - ./volume/postgres/spkio/init-trace.sql:/docker-entrypoint-initdb.d/init-trace.sql:ro

      - ./config/postgres/spkio/postgresql.conf:/etc/postgresql.conf:ro
    restart: always
    networks:
      - private-network

  discovery:
    container_name: spkio-discovery
    build:
      context: ./spkio.discovery/

    image: spkio-discovery:latest
    # ports:
    #   - 8761:8761

      - private-network

      - redis
      - rabbitmq
      - db-spkio

  account:

      - spring.datasource.password=spkio

      - private-network

      context: ./spkio.auth.resource/
